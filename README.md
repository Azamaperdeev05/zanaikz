## 🎯 Жоба туралы

**ЗаңКеңес AI** — Қазақстан Республикасының заңнамасы бойынша жасанды интеллект технологияларын қолдана отырып, азаматтарға тегін құқықтық кеңес беруге арналған веб-платформа.

Платформа Apple/macOS стиліндегі минималистік дизайнмен жасалған, **SF Pro** шрифтін пайдаланады және пайдаланушыларға заңнамалық сұрақтар қою, заңнамалар базасын іздеу, заңгерлер тізімін қарау, құжат үлгілерін жүктеу және құқықтық калькуляторларды пайдалану мүмкіндігін береді.

> **Дипломдық жұмыс** — Бұл жоба дипломдық жұмыс аясында әзірленген.

## ⚙️ Конфигурация

### Орта айнымалылары (.env)

Жоба түбірінде `.env` файлын жасаңыз:

```env
# Firebase конфигурациясы
VITE_FIREBASE_API_KEY=your_firebase_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.firebasestorage.app
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id

# OpenRouter API
OPENROUTER_API_KEY=your_openrouter_api_key
OPENROUTER_MODEL=qwen/qwen3.6-plus:free
```

### Firebase баптау

1. [Firebase Console](https://console.firebase.google.com) → Жаңа жоба жасау
2. **Authentication** → Google Sign-In қосу
3. **Cloud Firestore** → Деректер қоры жасау
4. Firestore ережелерін жүктеу:

```bash
firebase deploy --only firestore:rules
```

### Firestore индекстері

```bash
firebase deploy --only firestore:indexes
```

## 🔒 Қауіпсіздік

### Firestore Security Rules

- **Пайдаланушылар**: Тек өз профилін оқу/жаңарту мүмкін
- **Чат сессиялары**: Тек жасаушы оқу/жаңарту/жою мүмкін
- **Хабарламалар**: Аутентификацияланған пайдаланушылар оқу/жасау мүмкін
- **Заңнамалар**: Тек оқу (жазу — тек админ)
- **Админ**: `azamaperdeev05@gmail.com` — толық қол жеткізу

### API қауіпсіздігі

- OpenRouter API кілті `.env` файлында сақталады
- Firebase API кілттері `VITE_` префиксімен (Firestore Rules арқылы қорғалған)
- Google OAuth 2.0 аутентификация

### Орта айнымалыларын баптау

Firebase Hosting `.env` файлын оқымайды. Vite `VITE_` префиксті айнымалыларды build уақытында inline жасайды, ал `OPENROUTER_*` айнымалылары `process.env` арқылы Vite `define` конфигурациясы арқылы inject етіледі (`vite.config.ts` ішінде).
