# Жобаның архитектурасы мен құрылымы

Жоба заманауи Single Page Application (SPA) құрылымына (React 19) негізделген және масштабтауға өте ыңғайлы.

## 1. Папкалар құрылымы (File Tree)
```text
src/
├── components/       # Қайта қолданылатын UI компоненттері (Layout, AuthModal т.б.)
├── data/             # Статикалық деректер (mock data: заңгерлер, үлгілер, АЕК мөлшері)
├── pages/            # Негізгі беттер (Chat, Profile, Laws, Calculators, SOS)
├── store/            # Zustand глобальды күй менеджерлері (authStore, langStore)
├── utils/            # Қосалқы функциялар (i18n.ts, firebase.ts - баптаулар)
├── App.tsx           # Роутинг (React Router) кестесі
└── index.css         # Tailwind стилдері және глобаль шрифтер
```

## 2. Негізгі Архитектуралық Шешімдер

### 2.1. Component-Based Architecture
Әрбір бет (мысалы, `Calculators.tsx` немесе `Laws.tsx`) өзінің ішкі логикасымен оқшауланған. Бұл басқа әзірлеушіге белгілі бір модульді оңай түсініп, редакциялауға мүмкіндік береді. Ортақ навигация мен меню `Layout.tsx` арқылы барлық беттерді орап (wrap) тұрады.

### 2.2. State Management (Zustand)
Redux-тің ауырлығынан қашу үшін жеңіл әрі жылдам **Zustand** пайдаланылған:
- `authStore`: Пайдаланушының жүйеге кірген мәртебесін (Session), Firebase UID мен профиль мәліметтерін бақылайды.
- `langStore`: Қосымшаның интерфейс тілін (Қазақша/Орысша) басқарады.

### 2.3. Internationalization (Оқшаулау)
Толық i18n (Internationalization) тәсілі `utils/i18n.ts` ішінде жүзеге асқан. Қосымша екі тілді (kk, ru) жүйелі түрде ауыстырып, Firebase сұрауларын да тиісті тілде жіберетіндей етіп жобаланған.

### 2.4. Styling (Tailwind CSS)
Стильдер үшін толығымен Tailwind CSS 4 қолданылады. Бұл CSS файлдарын көбейтпестен, тікелей компонент деңгейінде дизайнды реттеуге, түстер палитрасын басқаруға көмектеседі. Глобальды түстер: Негізгі қөк (`#0071e3`), тақырыптар қарасы (`#1d1d1f`), сұр фонда `bg-[#f5f5f7]`.
