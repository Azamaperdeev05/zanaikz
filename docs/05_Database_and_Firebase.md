# Firebase және Дерекқор логикасы (DB & Auth)

Қосымшаның серверлік және дерекқор бөлігі толығымен **Google Firebase** экожүйесіне негізделген.

## 1. Аутентификация (Firebase Authentication)
`utils/firebase.ts` арқылы Firebase SDK қосылған.
- **Email/Password:** Пайдаланушы пошта мен құпия сөз арқылы тіркеліп, кіре алады.
- **Синхрондау:** `authStore.ts` ішіндегі `onAuthStateChanged` тыңдаушысы қолданушының жүйеге кіргенін сезіп, автоматты түрде жағдайды (state) жаңартып отырады.

## 2. Firestore Дерекқоры (NoSQL Database)
Пайдаланушы деректері мен чат тарихы `Cloud Firestore` ішінде сақталады.

### Коллекциялар құрылымы:
1. **`users/{uid}`**
   - Пайдаланушының жеке мәліметтері саналады (аты, тегі, телефон нөмірі, таңдаған аймағы).
   - `Profile.tsx` бетіндегі мәліметтер осы құжаттан оқылады және осында жаңартылады (`setDoc`, `updateDoc`).

2. **`chats/{uid}/sessions/{sessionId}`**
   - **`sessionId`**: Әрбір жаңа чат ашылған сайын генерацияланатын уникалды ID.
   - **Құрамы:** `title` (Чаттың қысқаша тақырыбы), `createdAt` (құрылған уақыты), `updatedAt` (соңғы хабарлама уақыты).

3. **`chats/{uid}/sessions/{sessionId}/messages/{messageId}`**
   - Бұл — нақты диалогтар коллекциясы.
   - **Құрамы:**
     - `role`: 'user' (пайдаланушы сұрағы) немесе 'model' (AI жауабы).
     - `content`: Хабарламаның мәтіні (Markdown форматында болуы мүмкін).
     - `timestamp`: Уақыт белгісі (ServerTimestamp).

## 3. Real-time Listeners (Нақты уақыттағы тыңдау)
`Chat.tsx` файлында `onSnapshot` функциясы қолданылады. Бұл пайдаланушы чатты ашқан кезде немесе AI-дан жаңа жауап келгенде, дерекқордағы мәліметті оқып, бетті (UI) автоматты түрде қайта жүктеусіз жаңартуды қамтамасыз етеді.

## 4. Қауіпсіздік ережелері (Firestore Security Rules)
Әрбір пайдаланушы тек өзінің `uid` (идентификаторымен) сәйкес келетін құжаттарды ғана оқып, жаза алатындай етіп Firebase консолінде Security Rules бапталған (Missing or insufficient permissions қателігін болдырмау үшін).
